CHECKOUT TEST CASES

Module: Cart, checkout summary, order placement, order retrieval, purchase retrieval, and status updates

TC-CO-01
Title: Get cart for authenticated user
Preconditions: Valid user token
Steps:
1. Send GET /api/checkout/cart with Authorization Bearer token.
Expected Result:
- 200 response returned.
- Only the logged-in user's cart items are returned.

TC-CO-02
Title: Reject cart access without authentication
Preconditions: Backend running
Steps:
1. Send GET /api/checkout/cart without Authorization header.
Expected Result:
- 401 Unauthorized returned.

TC-CO-03
Title: Add cart item with valid payload
Preconditions: Valid user token
Steps:
1. Send POST /api/checkout/cart/items with valid itemName, quantity, size, and unitPrice.
Expected Result:
- 201 response returned.
- Item is added to the current user's cart.

TC-CO-04
Title: Reject cart item with invalid quantity
Preconditions: Valid user token
Steps:
1. Send POST /api/checkout/cart/items with quantity 0 or a negative value.
Expected Result:
- 400 response returned.
- Validation message is returned.

TC-CO-05
Title: Merge duplicate cart item by same name and size
Preconditions: Valid user token, matching cart item already exists
Steps:
1. Add the same item again with the same size.
Expected Result:
- Existing cart item quantity increases.
- Duplicate row is not created.

TC-CO-06
Title: Update cart item quantity
Preconditions: Valid user token, existing cart item ID
Steps:
1. Send PATCH /api/checkout/cart/items/:cartItemId with quantity greater than 0.
Expected Result:
- 200 response returned.
- Cart item quantity is updated.

TC-CO-07
Title: Reject update for non-existent cart item
Preconditions: Valid user token
Steps:
1. Send PATCH /api/checkout/cart/items/:cartItemId using a fake ID.
Expected Result:
- 404 response returned.
- Message indicates cart item not found.

TC-CO-08
Title: Remove cart item
Preconditions: Valid user token, existing cart item ID
Steps:
1. Send DELETE /api/checkout/cart/items/:cartItemId.
Expected Result:
- 200 response returned.
- Removed item no longer appears in the user's cart.

TC-CO-09
Title: Generate checkout summary for Delivery order
Preconditions: Valid user token, cart contains items
Steps:
1. Send POST /api/checkout/summary with orderType Delivery.
Expected Result:
- 200 response returned.
- pricing object is returned.
- cartItems are returned.
- lastDeliveryDetails are returned.

TC-CO-10
Title: Generate checkout summary for Takeaway order
Preconditions: Valid user token, cart contains items
Steps:
1. Send POST /api/checkout/summary with orderType Takeaway.
Expected Result:
- 200 response returned.
- pricing object is returned without delivery failure.

TC-CO-11
Title: Reject invalid checkout order type
Preconditions: Valid user token
Steps:
1. Send POST /api/checkout/summary or /api/checkout/place-order with orderType set to an unsupported value.
Expected Result:
- 400 response returned.
- Message indicates orderType must be Delivery or Takeaway.

TC-CO-12
Title: Reject place order when cart is empty
Preconditions: Valid user token, no items in cart
Steps:
1. Send POST /api/checkout/place-order with a valid checkout payload.
Expected Result:
- 400 response returned.
- Message indicates the cart is empty.

TC-CO-13
