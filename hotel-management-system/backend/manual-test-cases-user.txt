TC-CO-06-U
Title: Update cart item quantity (User)
Preconditions: Valid user token, existing cart item ID
Steps:
1. Send PATCH /api/checkout/cart/items/:cartItemId with quantity > 0 (e.g., { "quantity": 2 }) using Authorization: Bearer <userToken>.
Expected Result:
- 200 response returned.
- Cart item quantity is updated for the given cartItemId.

TC-US-01
Title: User signup
Preconditions: Email is not registered, email is not the admin email
Steps:
1. Send POST /api/auth/signup with valid payload (fullName, email, password, phone, streetAddress1, streetAddress2, cityTown).
Expected Result:
- 200 response returned.
- success=true returned with token and user details.

TC-US-02
Title: User signup with existing email
Preconditions: Email is already registered
Steps:
1. Send POST /api/auth/signup using an already-registered email.
Expected Result:
- 400 response returned.
- Email already registered error is returned.

TC-US-03
Title: User login (local)
Preconditions: User account exists with password login enabled (passwordHash exists)
Steps:
1. Send POST /api/auth/login with valid email and password.
Expected Result:
- 200 response returned.
- success=true returned with token and user details.

TC-US-04
Title: User login with invalid credentials
Preconditions: User account does not exist OR password is incorrect
Steps:
1. Send POST /api/auth/login with invalid email/password.
Expected Result:
- 401 response returned.
- Invalid credentials message is returned.

TC-US-07
Title: Update user profile (me)
Preconditions: Valid user token (role=user)

1. Send PUT /api/users/me with Authorization: Bearer <userToken> and valid payload (fullName, phone, streetAddress1, streetAddress2, cityTown).
